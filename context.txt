1. Core Technology Stack

To achieve high-quality rendering (satisfying zooms, dynamic TikTok-style subtitles) entirely locally, the tech stack must be modern and Python-centric.

    CLI User Interface: Rich (for beautiful terminal styling, progress bars, and colored outputs) + InquirerPy (for interactive terminal menus and file selection).

    Audio Transcription (Word-level): faster-whisper. It runs locally, is highly optimized, and most importantly, outputs word-level timestamps which are mandatory for kinetic subtitles and cutting out "uhms".

    Local AI Engine: Ollama running the llama3:8b or phi-3 model via the ollama Python package. It will analyze the transcript locally to find viral moments and score them without API costs.

    Video Processing & Editing Engine: MoviePy (v2) for rendering, text overlays (using Pillow via MoviePy v2's built-in TextClip), video cutting (subclipped/concatenate_videoclips), and programmed zooms. Use Python's subprocess module to call ffmpeg directly for audio extraction only (the ffmpeg-python package is unmaintained since 2019 and has known security vulnerabilities).

    Silence & Dead Air Removal: Determined purely from faster-whisper's word-level timestamps. (If there is a gap > 1.5 seconds between the end of one word and the start of the next, it's flagged as dead air. No extra audio library needed).

3. Step-by-Step Execution Flow (The Process Doc)

Provide this exact logic flow to your AI agent so it understands how the modules interact.
Phase 1: Environment & UI Initialization

    Boot Up: The script runs main.py.

    Environment Check: Use os and pathlib to check if Raw/ and Clips/ exist in the current working directory.

        If missing, use Rich to prompt the user: "Required folders missing. Create them? (Y/n)". If yes, generate them.

    File Discovery: Scan the Raw/ directory for .mp4 and .mkv files.

    User Selection: Use InquirerPy to present a selectable list of the found videos. The user picks one to process.

Phase 2: Transcription & Dead Air Detection

    Audio Extraction: Use subprocess to call ffmpeg directly to extract the audio track as a .wav file (this speeds up transcription). Example: subprocess.run(["ffmpeg", "-i", input_path, "-vn", "-acodec", "pcm_s16le", "-ar", "16000", "-ac", "1", wav_path]).

    Faster-Whisper Processing: Pass the .wav to faster-whisper.

        Agent Instruction: Ensure word_timestamps=True is enabled.

    Dead Air Mapping: Iterate through the word timestamps.

        Rule 1: If a word is "um", "uh", "uhm", "erm", or "hmm" (case-insensitive), mark its timestamp for deletion. (Do NOT delete "like" — it is commonly used as a meaningful word in context. Only true filler words that can be unconditionally removed. Note: Whisper typically outputs "um" not "uhm" for English, so check for both.)

        Rule 2: If the gap between the end of Word A and the start of Word B is greater than 1.5 seconds (streamer silently reading chat), mark that time range for deletion.

Phase 3: AI Analysis & Viral Scoring (Ollama)

    Chunking: Group the transcript into 60-to-90-second logical chunks.

    The Prompt: Send the text chunk (with timestamps included, e.g. "[00:12.340 -> 00:12.800] Hello") to the local Ollama LLM with a strict system prompt:

        "You are a viral TikTok editor. Read this transcript segment with timestamps. Identify if there is a highly entertaining, stupid, violent, or emotional moment. If yes, rate its virality out of 20. Return your response STRICTLY as a JSON object containing: { 'is_clip': true/false, 'score': 1-20, 'reason': 'short description', 'start_time': <float seconds>, 'end_time': <float seconds> }. The start_time and end_time must be actual timestamp values from the transcript, NOT word indices."

    Filtering: The Python script parses the JSON. Only chunks with a score of 15/20 or higher are passed to the editing engine.

        Agent Instruction: Small local LLMs (8B params) frequently produce malformed JSON (wrong quotes, missing commas, extra preamble text). Implement robust JSON parsing: (1) Try json.loads() first, (2) If that fails, extract the JSON substring using regex (find the first '{' to the last '}'), (3) If still failing, strip common issues (smart quotes -> straight quotes, trailing commas), (4) If all parsing fails, log a warning and skip that chunk rather than crashing.

Phase 4: The Editing Engine (MoviePy)

This is the most complex part for the AI to write. Break it down into these exact sub-tasks for the agent:

    Base Cutting & Pacing:

        Extract the video segment using the LLM's chosen start and end timestamps.

        Apply the "Dead Air Map" created in Phase 2. IMPORTANT: The dead air map uses timestamps from the full video. Before applying to the extracted clip, offset all dead air timestamps by subtracting the clip's start_time (clip_start = LLM's start_time). Only keep dead air ranges that fall within the clip's time range. Use MoviePy v2's clip.subclipped() (NOTE: In MoviePy v2, the old v1 method subclip() was renamed to subclipped()) to cut out the dead air segments, then concatenate_videoclips to stitch together only the speaking parts, resulting in an "aggressive pacing" jump-cut style.

    Satisfying Zooms (Keyframing):

        Agent Instruction: To create a digital zoom, use MoviePy v2's clip.cropped() and clip.resized() methods combined with a time-based easing function. (Note: In MoviePy v2, the old v1 methods crop/resize were renamed to cropped/resized.)

        Apply a slow 110% zoom-in during the first 3 seconds to hook the viewer.

        Trigger a sudden 120% punch-in zoom immediately after each jump-cut point (the boundaries between concatenated dead-air-removed segments — these are known from the dead air removal step, not "detected" after the fact) to emphasize a punchline or reaction.

    Dynamic Subtitles:

        Take the word-level timestamps from Whisper.

        Generate a TextClip for every 1-3 words.

        Style requirements: Use a bold font file path (e.g., a locally installed Montserrat Black or Impact .ttf/.otf file — MoviePy v2 requires an explicit font file path, not just a font name). Yellow/white text with a thick black stroke (stroke_color='black', stroke_width=3+). Center them on the Y-axis (vertical video style).

    Output Format:

        The final output must be vertical (9:16 aspect ratio, e.g. 1080x1920) for TikTok/Reels/Shorts. Before applying zooms and subtitles, the editor must crop or pad the source video to 9:16. If the source is 16:9, center-crop the middle third. If the source is already 9:16, use as-is. The LLM-selected clip should be centered on the main subject/action when cropping.

Phase 5: Rendering and Output

    Folder Creation: Create a sub-folder inside Clips/ named after the source video (e.g., Clips/Stream_VOD_01/).

    Exporting: Use MoviePy to write the final .mp4. Set preset='medium' (balances quality and speed — 'ultrafast' produces large files with lower quality, contradicting the high-quality rendering goal) and threads=4 (or more) for faster rendering.

    Metadata File: Write a [Clip_Name]_meta.txt file next to the video containing the virality score out of 20 and the LLM's generated description of why it's entertaining.

    Cleanup: Delete the temporary .wav files.
